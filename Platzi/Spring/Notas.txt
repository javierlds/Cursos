********************* ANOTACIONES JPA*************************************************************************************************************
anotaciones para conectar clases con BD y evitar forma nativa

	@Entity  => indica clase java que representa tabla de la bd, clases que mapearan las tablas
	@Table => recibe el nombre de la tabla que mapea la clase
	@Column => anotacion que recibe el atributo de nuestra clase, se pone solo cuando el nombre de nuestra columna sea diferente al atributo de la tabla
	@Id  & @EmbededId => representa clave primaria , EmbededId Es cuando es llave compuesta
	@GeneratedValue =>  es una anotacion que permite genera valores automaticamente para nuestras tablas
	@OneToMany & @ManyToOne  => representa relaciones en nuestras clases 
	@OneToMany(mappedBy) => relacion que va de uno a muchos. En mappedBy se debe especificar el nombre de la variable privada asociada a la clase que se relaciona
	@ManyToOne => relacion que va de muchos a uno
	
***********************************SPRING DATA   ***********************************************************************************	
	------------- 
	Proyecto que usa JPA para la gestion de tareas desde JAVA sea mas poderoza
	
		SUB PROYECTOS:
			- Spring Data JPA => - Bases de datos relacionales
								 - optimizar tareas repetitivas
								 - Repositorios sin codigo, permite hacer operaciones en CRUD
								 - Auditorias transparentes
								 
								 Repositorios sin codigo JPARepository, CrudRepository & PagingAndSortingRepository
								 Auditorias Transparentes(cuando se inserto, actualizo, elimino)
	
					

		TIPOS DE REPOSITORIO:
			* CrudRepository:  operaciones CRUD 
			* PagingAndSortingRepository: hacer todo el CRUD , paginacion y ordenamiento
			* JPARepository: hacer todo lo de CrudRepository y PagingAndSortingRepository, pero adicionalmente FLUSH, combina o 
			                 guarda todo en memoria sin que otras entidades o entorno vean esos cambios en la bd
